# Security policy

Do not open a public issue containing a vulnerability, access token, private
memory, source excerpt, or account detail. Use GitHub's private vulnerability
reporting for this repository. If that option is unavailable, contact the
repository owner privately through the verified contact shown on GitHub.

Include the affected client, reproduction steps, and impact. Redact all tokens
and private brain content. The hosted OAuth and hub implementation are kept in
a separate private repository; this public repository contains no service
credentials or deployment configuration.
